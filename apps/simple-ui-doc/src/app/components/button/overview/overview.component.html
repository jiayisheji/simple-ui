何时使用

标记了一个（或封装一组）操作命令，响应用户点击行为，触发相应的业务逻辑。

原生的 <button> 和 <a> 元素总是用于为用户提供最直接和可访问的体验。当执行某些操作时，应该使用 <button> 元素。当用户导航到另一个视图时，应该使用 <a> 元素。

三种不同的类型

按钮允许应用程序传达操作并指导用户意图。

填充按钮看起来很厚重。它们将用户的注意力转移到应用程序建议用户执行的主要操作上。
轮廓按钮提供了一种更轻的按钮样式。当页面上有许多同等重要的动作时，它们被用来表示一个次要操作或辅助主要操作，以减少视觉干扰。
文字按钮用于多种场景，它们被用作第三级按钮。它们还可以内联使用，因为它们在样式上与内容不同，可以作为内容旁边的按钮来识别。

文字按钮 VS 链接按钮

当期望用户执行操作时，请使用文字按钮。

链接按钮

当希望将用户带到其他页面时，请使用链接按钮。

摆放位置

关于按钮的放置，有两种不同的模式。

Z 模式

Z模式是用户浏览受限容器中内容的自然方式，并且当任务从左上角定向并以容器的右下侧的主要号召性用语结束时。常用于对话框、向导等。

F 模式

F模式是在不受限制的容器（例如页面本身上的表单）中浏览内容的自然方法。用户将逐行浏览内容，最后到达一个操作调用。常用于表单、卡片等。

颜色和排版

一致的按钮样式使用户更容易识别在整个应用程序中采取操作的区域。

按钮内的英文文本始终为大写。通过将按钮文本与页面上的链接和其他内容区分开来指示操作。在与用户意图有关的按钮上使用描述性语言。

按钮的边框半径为4px。

三种按钮尺寸

按钮的高度会和输入框、选择框相关联，当它们同时出现在表单中时。

较大按钮一般都是相对独立使用，用来增强号召性用语的内容区域。
默认按钮一般用主要操作。
小型按钮一般用比较次级操作，用来减少号召性用语的内容区域。

当 <a> 应用按钮样式时，风格会和 <button> 保持一致，当应用链接样式时，高度与行高由父级决定。

基于操作的颜色

根据操作结果的严重程度，可以使用不同的颜色。例如，在删除文件时使用红色按钮表示高度危险。

主题色提供了应用程序之间的一致性。它训练用户在寻找操作时寻找那个颜色。这个操作主题色可以在所有类型的按钮、选项卡和其他与操作相关的组件中找到。

如果在一组按钮中使用颜色强调操作，将主要操作安排在左侧，其次是次要操作，并将警告操作作为组中的最后一个按钮。

图标按钮

图标按钮在界面空间有限的地方很有用。如果一个图标能很好地表示操作意图，用户有时就能比阅读文本更快地识别它们。最后，使用图标按钮可以帮助解决使用长标签困难的情况。

选择一个最能描述用户将要执行的操作的图标。用户可以避免使用未知或抽象的图标按钮。

如果只使用图标按钮时，建议将 title 属性添加到的图标按钮。这为不熟悉您的图标按钮可能产生什么操作的用户增加了一些额外的上下文。将鼠标悬停在图标按钮上一会儿会显示带有标题文本的工具提示。文案应反映正在完成的操作。

如果有足够的空间，添加文本可帮助用户理解操作。使用图标启动图标按钮，然后输入文本。

辅助功能样式

虚线边框是轮廓按钮的变种，以虚线呈现

充满按钮将使按钮适合其父宽度。

胶量按钮加大圆角，使按钮看起更加圆润

圆角按钮是图标按钮的变种，以圆形呈现

可交互状态加载中

默认按钮是没有交互行为，给按钮扩充加载状态，当按钮进行异步操作时，加载状态很好给用户一个提示交互效果

按钮分组

默认按钮无间隙

通过添加 spaced 属性增加间隙
